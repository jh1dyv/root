
submodule の指す先が別途更新されたときにリンク先を追随させる方法

※ 下図で、<top>/buildtool, <top>/dependency で直接変更を push した場合
　 <top>/Springhead/<buildtool> などで変更をした場合については未だ不明

※ commit 時のコメントに日本語が入っていると、dos窓では表示が出来ずに見難い
　 Git Bash を使うとよい

directory 構成は次のとおり
<top> --+-- buildtool/
        +-- Springhead --+-- <buildtool>
        |                +-- core
        |                +-- <dependency>
        |                +-- .gitmodules
        +-- dependency/

<buildtool>, <dependency> は submodule update された状態とする

================================================================
<buildtool> についての例（<dependency> についても同様）

・最新の commit を調べる

> cd <top>/buildtool
> git log --oneline --max-count=1

    444f9dc (HEAD -> master, origin/master, origin/HEAD) pythonlib は src/RunSwig に移動

・リンク先の commit を 最新のもの（444f9dc）に変更する

> cd <top>/Springhead/buildtool
> git pull origin HEAD
> git checkout 444f9dc

    HEAD is now 444f9dc... pythonlib は src/RunSwig に移動
