{\bf Text mode:}

三角括弧は、typewriter fontを使って {\tt <}tag?{\tt >} とするか、
math modeにして $<$tag?$>$ とする。
さもないと <tag?> のように字化けする。

アンパサントは、\& のようにエスケープしないとエラーとなる。

Table環境では、エスケープなしのアンパサントを使用する。

\medskip
\begin{tabular}{|c|c|}\hline
	{\bf 項目1} & {\bf 項目2} \\\hline
	value 1	& value 2 \\\hline
\end{tabular}

\bigskip
\noindent
{\bf verbatim environment:}

原則としてすべてそのまま書けば良いが、
{\tt \#include <...>} だけは例外で、
三角括弧はそのまま書く。
ここに最初から \BS textless, \BS textgreater を使うと、
この文字列がそのまま表示されてしまう
(これらは\LaTeX のマクロではない)。
\begin{verbatim}
#include <something.h>                       // < > をそのまま記述
#include \textless something.h\textgreater   // マクロで記述
\end{verbatim}

\bigskip

{\bf sourcecode environment:}

\end{sourcecode}
