\section{DailyBuildとは}
\label{sec:WhatIsDailyBuild}

Springheadに加えられた修正、新規に組み込まれたソースと既存ソースとの整合性の確認、
または何らかの理由によるソース劣化の防止等を目的として、
次の処理をWindowsタスクとして日々(自動的に)実行している。

これらのタスクはunix上でも実行可能である(一部未実装の部分がある)。
確認に使用したunixは、ubuntu--ja--16.04--desktop--amd64 である。

\begin{enumerate}
  \item	GitHubから最新バージョンを取り出し、
	stubおよびspringhead libraryを作成する。
	\vspace{.5ex}
	\label{BuildEnvironment}
	\begin{narrow}[\WID]{\small
	    \begin{tabular}{|l|l|} \hline
	    	windows &	Visual Studio 2015　x64/Release構成 \\\hline
	    	unix    &	{\tt g++　-O2} \\\hline
	    \end{tabular}
	}\end{narrow}
	\medskip
  \item	\tt{src/tests}以下にあるテストソリューションに対してビルド＆テストを実行する。
	環境は\ref{BuildEnvironment}.と同じ。
  \item	\tt{src/Samples}以下にあるテストソリューションに対してビルドを行なう。
	環境は\ref{BuildEnvironment}.と同じ。
  \item	その結果(ログファイル、ライブラリファイルおよびドキュメントファイル)を
	サーバへ登録する。
  \item	生成した最新ドキュメン(pdf/html)をWikiで公開する。
\end{enumerate}

\noindent
この一連のタスクをDailyBuildと呼ぶ。

\medskip
また、DailyBuildの結果をまとめたものWikiに反映させるためのタスクも日々実行している。
このタスクをMakeReportと呼ぶ。
