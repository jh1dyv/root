\section{DailyBuildとは}
\label{sec:WhatIsDailyBuild}

\noindent
Springheadに加えられた修正、新規に組み込まれたソースと既存ソースとの整合性の確認、
又は何らかの理由によるソース劣化の防止等を目的として、
次の処理をWindowsタスクとして日々(自動的に)実行している。

これらのタスクはunix上でも実行可能である
(確認に使用したのはubuntu--ja--16.04--desktop--amd64
　ただし一部未実装の部分がある)。

\begin{enumerate}
  \item	\label{BuildEnvironment}
	GitHubから最新バージョンを取り出し、
	stubおよびspringhead libraryを作成する。
	\vspace{-.5\baselineskip}
	\small{\begin{Table}[r][4em]{|l|l|}\hline
		\Item{Windows}{Visual Studio 2015, x64/Release構成}\hline
		\Item{unix}{\tt{g++, -O2}}\hline
	\end{Table}}

  \item	\tt{src/tests}以下にあるテストソリューションに対して
  	ビルド＆テストを行なう。\\
	環境は\ref{BuildEnvironment}.と同じ。

  \item	\tt{src/Samples}以下にあるテストソリューションに対して
  	ビルドを行なう。\\
	環境は\ref{BuildEnvironment}.と同じ。

  \item	その結果(ログ、ライブラリ及びドキュメント)をサーバへ登録する。

  \item	生成した最新ドキュメン(pdf/html)をWikiで公開する。
\end{enumerate}

この一連のタスクをDailyBuildと呼ぶ。
また、DailyBuildの結果をまとめたものWikiに反映させるためのタスクも
日々実行している。
このタスクをMakeReportと呼ぶ。
