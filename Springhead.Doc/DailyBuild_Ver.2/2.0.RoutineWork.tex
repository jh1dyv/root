\section{ルーチン作業}
\label{sec:RoutineWork}

\noindent
DailyBuild の結果を Wiki (\tt{http://springhead.info/wiki/index.php?devel}) で確認する。

\begin{enumerate}
  \item	次の2つのリンクを辿り、以前の結果と相違がないか確認する。
	\begin{itemize}
	  \item	開発版 − 今朝のビルド − 今朝のビルドの状況(過去のビルドの履歴)
	  \item	開発版 − 今朝のビルド − Samplesのビルドの状況(過去のビルドの履歴)
	\end{itemize}

  \item	相違があったときは「今朝のビルドのログ」等を見て原因を特定する。
	\begin{itemize}
	  \item	テスト系の不具合のときは修正をする。
	  \item	開発系の不具合のときは開発者に連絡をし、必要なら対処を依頼する。
		\begin{narrow}
			メール先： springhead開発者 \verb|<dev@springhead.info>|
		\end{narrow}
		さらにバグトラッカへ登録も検討する(最近は活用されていない)。
		\begin{narrow}
		    リンク： Wiki開発者向け情報のページにある(ポップアップ画面に注意)
		\end{narrow}
		\begin{description}
		  \item[※]
			今朝のビルドが最後まで実行できていない場合には、
			DailyBuildを実行するマシン\tt{DESKTOP-KD3C7HS(192.168.91.194)}の
			\begin{narrow}
				\verb|I:/DailyBuild/DailyBuild.log| \\
				\verb|I:/DailyBuild/MakeReport.log|
			\end{narrow}
			を見て原因を特定する必要がある。
		\end{description}
	\end{itemize}

  \item	週に1回程度は次も確認する。
	\begin{enumerate}
	  \item	ドキュメントページからAPIリファレンス のリンクを辿り、
		ページが正しく作成されているか確認する。
		特に、ヘッダファイルのインクルード依存関係図が生成されているか確認する。
		(例えば、[ユーティリティクラス]--[\tt{BaseUtility.h}]と辿ってみる)
	  \item	ドキュメントページからWindows Helpファイルをダウンロードし、
		上記と同様に内容を確認する。
		ダウンロードしたファイル(\FILE{Springhead.chm})は、
		プロパティウィンドウを開き、
		セキュリティフィールドの「ブロックの解除」を行なわないと
		内容が表示されないので注意すること。

		\bf{[注意]} 現在Helpファイルは作成していない。
	\end{enumerate}
\end{enumerate}

\medskip
\noindent

``\ref{sec:Script} スクリプトの説明''を参照のこと。
